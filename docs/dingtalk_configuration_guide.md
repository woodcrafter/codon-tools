# 钉钉集成配置指南

本文档说明如何配置钉钉OAuth登录和消息推送功能。

## 前提条件

- 拥有钉钉企业管理员权限
- 已部署基因合成平台到公网可访问的域名

## 一、钉钉OAuth登录配置

### 1. 创建钉钉企业内部应用

1. 登录[钉钉开发者后台](https://open-dev.dingtalk.com/)
2. 进入"应用开发" → "企业内部开发" → "创建应用"
3. 填写应用信息：
   - 应用名称：基因合成平台
   - 应用描述：企业内部基因设计与订单管理系统
   - 应用图标：上传应用logo

### 2. 配置应用权限

在应用详情页面，配置以下权限：

**接口权限**：
- `Contact.User.Read`：获取用户基本信息
- `openid`：获取用户OpenID
- `corpid`：获取企业ID

### 3. 配置OAuth回调域名

在"开发配置" → "登录与分享" 中：

1. 启用"扫码登录"和"密码登录"
2. 配置回调域名：
   ```
   https://your-domain.com/api/oauth/dingtalk/callback
   ```
   **注意**：必须是HTTPS协议，且域名已备案

3. 配置应用首页地址：
   ```
   https://your-domain.com
   ```

### 4. 获取应用凭证

在应用详情页面，获取以下信息：
- **AppKey (ClientId)**：应用的唯一标识
- **AppSecret (ClientSecret)**：应用密钥

### 5. 配置环境变量

在Manus平台的环境变量设置中，添加以下配置：

| 环境变量名 | 说明 | 示例值 |
|-----------|------|--------|
| `DINGTALK_APP_KEY` | 钉钉应用AppKey | `dingxxxxxxxxxxxxxx` |
| `DINGTALK_APP_SECRET` | 钉钉应用AppSecret | `xxxxxxxxxxxxxxxxxxxxxxxx` |
| `VITE_DINGTALK_APP_KEY` | 前端使用的AppKey（与DINGTALK_APP_KEY相同） | `dingxxxxxxxxxxxxxx` |

### 6. 测试登录流程

1. 访问平台首页
2. 点击"钉钉账号登录"按钮
3. 使用钉钉扫码或密码登录
4. 授权后自动跳转回平台

## 二、钉钉消息推送配置

### 1. 创建钉钉群机器人

1. 在钉钉中创建一个群聊（例如："基因平台订单通知群"）
2. 点击群设置 → 智能群助手 → 添加机器人 → 自定义
3. 填写机器人信息：
   - 机器人名称：基因平台通知
   - 消息推送：勾选"自定义关键词"或"加签"（推荐加签）

### 2. 配置安全设置

**方式一：加签（推荐）**

1. 选择"加签"安全设置
2. 复制生成的密钥（SEC开头的字符串）
3. 配置环境变量：
   ```
   DINGTALK_WEBHOOK_SECRET=SECxxxxxxxxxxxxxxxxxxxxxx
   ```

**方式二：自定义关键词**

1. 选择"自定义关键词"
2. 添加关键词：`订单`、`通知`、`基因平台`
3. 消息中必须包含至少一个关键词才能发送成功

### 3. 获取Webhook URL

创建机器人后，复制Webhook地址，格式如下：
```
https://oapi.dingtalk.com/robot/send?access_token=xxxxxxxxxxxxxxxx
```

### 4. 配置环境变量

在Manus平台的环境变量设置中，添加：

| 环境变量名 | 说明 | 示例值 |
|-----------|------|--------|
| `DINGTALK_WEBHOOK_URL` | 钉钉群机器人Webhook地址 | `https://oapi.dingtalk.com/robot/send?access_token=xxx` |
| `DINGTALK_WEBHOOK_SECRET` | 加签密钥（如果使用加签方式） | `SECxxxxxxxxxxxxxx` |

### 5. 测试消息推送

1. 创建一个测试订单
2. 检查钉钉群是否收到通知消息
3. 消息应包含订单号、项目名称、创建人等信息

## 三、通知场景说明

系统会在以下场景自动推送钉钉消息：

### 1. 订单创建通知
- **触发时机**：用户创建新订单时
- **通知对象**：订单通知群（所有成员）
- **消息内容**：
  - 订单号
  - 项目名称
  - 创建人
  - 创建时间
  - 订单详情链接

### 2. 订单状态变更通知
- **触发时机**：订单状态从一个阶段流转到另一个阶段
- **通知对象**：
  - 订单通知群
  - @指定负责人（如果已分配）
- **消息内容**：
  - 订单号
  - 项目名称
  - 状态变更（旧状态 → 新状态）
  - 当前负责人
  - 更新时间
  - 订单详情链接

### 3. 任务分配通知
- **触发时机**：管理员将任务分配给特定员工
- **通知对象**：@被分配的员工
- **消息内容**：
  - 订单号
  - 任务类型
  - 负责人
  - 截止时间
  - 订单详情链接

### 4. 实验结果上传通知
- **触发时机**：员工上传实验结果
- **通知对象**：订单通知群
- **消息内容**：
  - 订单号
  - 实验类型
  - 实验结果（成功/失败）
  - 操作人
  - 备注信息
  - 订单详情链接

### 5. 系统告警通知
- **触发时机**：系统检测到异常或重要事件
- **通知对象**：
  - 订单通知群
  - @所有人（如果是错误级别）
- **消息内容**：
  - 告警级别（信息/警告/错误）
  - 告警标题
  - 详细信息
  - 发生时间

## 四、常见问题

### Q1: 点击"钉钉账号登录"没有反应？

**可能原因**：
- 未配置钉钉OAuth凭证
- AppKey配置错误

**解决方法**：
1. 检查环境变量 `DINGTALK_APP_KEY` 和 `VITE_DINGTALK_APP_KEY` 是否正确配置
2. 确认AppKey与钉钉开发者后台显示的一致
3. 清除浏览器缓存后重试

### Q2: 登录后提示"授权失败"？

**可能原因**：
- AppSecret配置错误
- 回调域名未正确配置
- 应用权限不足

**解决方法**：
1. 检查 `DINGTALK_APP_SECRET` 是否正确
2. 确认回调域名配置为：`https://your-domain.com/api/oauth/dingtalk/callback`
3. 检查应用是否已启用 `openid` 和 `corpid` 权限

### Q3: 钉钉群没有收到通知消息？

**可能原因**：
- Webhook URL配置错误
- 安全设置不匹配
- 消息格式不符合要求

**解决方法**：
1. 检查 `DINGTALK_WEBHOOK_URL` 是否正确
2. 如果使用加签方式，确认 `DINGTALK_WEBHOOK_SECRET` 配置正确
3. 如果使用关键词方式，确认消息中包含配置的关键词
4. 查看服务器日志中的错误信息

### Q4: 消息推送频繁失败？

**可能原因**：
- 触发了钉钉的频率限制
- Webhook URL已过期或被删除

**解决方法**：
1. 钉钉群机器人有频率限制（每分钟最多20条），避免短时间内大量推送
2. 检查机器人是否被删除，如已删除需要重新创建
3. 考虑使用消息合并或延迟推送策略

### Q5: 如何临时禁用钉钉通知？

**方法一**：删除或注释环境变量
- 删除 `DINGTALK_WEBHOOK_URL` 环境变量
- 系统会自动跳过消息推送，不会报错

**方法二**：在钉钉群中暂停机器人
- 进入群设置 → 智能群助手
- 找到对应机器人，点击"停用"

## 五、安全建议

1. **保护应用密钥**：
   - 不要将 AppSecret 和 Webhook Secret 提交到代码仓库
   - 使用环境变量管理敏感信息
   - 定期轮换密钥

2. **限制访问权限**：
   - 仅授予必要的API权限
   - 定期审查应用权限范围
   - 监控异常登录行为

3. **使用HTTPS**：
   - 回调URL必须使用HTTPS协议
   - 确保SSL证书有效

4. **监控与审计**：
   - 记录所有OAuth登录事件
   - 监控Webhook调用频率和失败率
   - 定期检查钉钉开发者后台的应用日志
